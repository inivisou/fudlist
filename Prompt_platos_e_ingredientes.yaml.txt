Prompt_platos_e_ingredientes yaml

Dada la lista completa de platos que aparece a continuación, genera un archivo **YAML** llamado `platos_e_ingredientes.yaml` con la siguiente estructura:

Plato:
  - id: <número correlativo>                     # 1, 2, 3 …
    Nombre del plato: "<nombre tal cual>"
    comida: <true|false>                         # true si la etiqueta incluye “Comida”
    cena: <true|false>                           # true si la etiqueta incluye “Cena”
    tipo: "<categoría>"                          # una palabra que describa el tipo principal del plato
                                                  # (p.ej. pasta, pescado, sándwich, legumbres, sopa, crema, guiso, etc.)
    Para_Eme: <true|false>                       # true si la etiqueta incluye “Eme”
    Para_Cris: <true|false>                      # true si la etiqueta incluye “Cris”
    Ingredientes: [<id‑ingrediente 1>, <id‑ingrediente 2>, …]
    enlace: "<URL del plato>"

Ingredientes:
  - id: <número correlativo>                     # 1, 2, 3 …
    nombre: "<nombre del ingrediente>"
    Supermercado: ""                              # opcional, lo dejamos vacío

---  

Reglas que debe aplicar el generador  

1. **Un solo registro por ingrediente** – aunque aparezca en varios platos, se crea **un único ID** y se reutiliza en todos los platos que lo necesiten.  
2. El campo **`tipo`** se determina a partir del nombre o la naturaleza predominante del plato (p.ej. si el nombre contiene “pasta”, “espaguetis”, “macarrones” → `pasta`; si contiene “sándwich” → `sándwich`; si la receta es esencialmente a base de legumbres → `legumbres`; si es una sopa o crema → `sopa` o `crema`; etc.).  
3. Los valores **`comida`** y **`cena`** son `true` cuando la etiqueta correspondiente está presente dentro de los corchetes `[Comida]` o `[Cena]`.  
4. Las etiquetas de usuarios (`Eme`, `Cris`) controlan los campos **`Para_Eme`** y **`Para_Cris`**.  
5. El campo **`enlace`** copia literalmente la URL que sigue a la etiqueta `[Enlace:]`.  
6. El resultado se escribe en formato **YAML** sin comentarios adicionales.
