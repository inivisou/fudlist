Quiero montar una aplicacion que me genere los menús que tengo que hacer para toda la semana.

No quiero usar base de datos, ya que va a ser en principio simple, se leen platos, ingrecientes y recetas, como no quiero hacer vulnerable mi SGBD ni mi servidor, he decidido hacerlo con ficheros yaml.

Es decir lo que serían tablas serán ficheros yaml, en los que tendran una estructura (que serían las columnas).

Los ficheros yaml serán Platos.yaml, Ingredientes.yaml, Recetas.yaml

El fichero "Platos.yaml" contendrá platos, que tendrá relacionada una receta del fichero "Recetas.yaml" y el fichero "Recetas.yaml" tendrá descripcion de la receta e ingredientes que se encontrarán en el fichero "Ingredientes.yaml"

Una vez dicho esto lo que es la estructura de datos será de la siguiente manera:

    - Ejemplo del fichero Platos.yaml

	 Plato:
	  - id: 1
	    Nombre del plato: "Sopa de cocido + Filete de lomo" # Nombre del plato
	    comida: true                                        # Si aplica al momento de la comida (la que suele ser a las 14:00)
	    cena: false  										# Si aplica al momento de la cena
	    tipo: sopa  										# Tipo principal del plato (pasta, legumbres, sopa, carne,pescado.....)
	    Para_Eme: true										# Si le gusta a Eme
	    Para_Cris: true										# Si le gusta a Cris
	    receta: receta.php?id=1 							# Enlace a la receta, siempre será la pagina receta.php?id=(Id del plato)

	  - id: 2
	    Nombre del plato: "Sopa de fideos + Filete de ternera"
	    comida: true                                        # Si aplica al momento de la comida (la que suele ser a las 14:00)
	    cena: false  										# Si aplica al momento de la cena
	    tipo: sopa  										# Tipo principal del plato (pasta, legumbres, sopa, carne,pescado.....)
	    Para_Eme: true										# Si le gusta a Eme
	    Para_Cris: true										# Si le gusta a Cris
	    receta: receta.php?id=2 							# Enlace a la receta, siempre será la pagina receta.php?id=(Id del plato)

    - Ejemplo del fichero Recetas.yaml

	 Receta:
	  - id: 1
	  	id_receta: 1
	    Titulo_html : "Receta de " + (titulo del plato)
	    Subtitulo_html : String
	    Texto_html: 
	    Ingredientes: [<id‑ingrediente 1>, <id‑ingrediente 2>, …]
	    Herramientas necesarias [tostadora,termomix,batidora....]										
	   	Enlace: "https://www.recetasderechupete.com/sopa-de-fideos/11389/"

	  - id: 2
	  	id_receta: 2
	    Titulo_html : "Receta de " + (titulo del plato)
	    Subtitulo_html : String
	    Texto_html: 
	    Ingredientes: [<id‑ingrediente 1>, <id‑ingrediente 2>, …]										
	   	Enlace: "https://www.recetasderechupete.com/sopa-de-fideos/11389/"

	- Ejemplo del fichero Ingredientes.yaml
	  - id: <número correlativo>                     # 1, 2, 3 …
	    nombre: "<nombre del ingrediente>"
	    calorias_x_100g:							#opcional
	    Supermercado: ""                            # opcional, lo dejamos vacío


Se deben genear 2 php:

- Primer php "index.php"
	- Éste php tendrá un selector multiple que sea "No Contenga Herramientas" y listar todas las herramientas que se encuentren en el fichero Recetas.yaml
	- Un campo input numerico entero con un mínimo de 1 y un máximo de (cantidad de platos que haya en Platos.yaml)/2, con el texto "Días"
	- Un boton "Generar" que generará tantos días como haya en el campo "Días"
	- El php generará:
		- Una tabla responsive, que se pueda ver en dispositivos moviles y en pantallas de ordenador de manera ágil y cumpliendo con los requisito WCAGG 2.2.
		- Esta tabla tendrá como cabecera
			Numero de plato | Añadir |Comida | Para | Añadir| Cena | Para
		- El ejemplo de dos lineas es el siguiente
			1 | [] Checkbox para seleccionar la comida 1 | Sopa de cocido + Filete de lomo [Receta (enlace a la receta del plato)]| Eme </br> Cris| [] Checkbox  para seleccionar la cena 1|Sándwich mixto con huevo [Receta (enlace a la receta del plato)] | Eme </br> Cris
			2 | [] Checkbox para seleccionar la comida 2 |Garbanzos con pisto [Receta (enlace a la receta del plato)]| Cris|Checkbox para seleccionar la cena 2 | Wrap de pollo [Receta (enlace a la receta del plato)] | Eme </br> Cris

	- El fichero index.php, debe ir haciendo una lista debajo que se recargara y ejectuara estilo AJAX (asincrono quiero decir)
		- Los platos que se vayan seleccionando y se cargaran sus ingredientes al lado , ejemplo a continuación.
			Día | Periodo | Ingredientes
			- El ejemplo de seleccionar en el caso anterior del primero la comida y del segundo la cena sería:
			1 | Comida | Sopa de cocido + Filete de lomo | · Filetes de lomo <br/> · Caldo de cocido  <br/> · Fideos  <br/> · Sal
			1 | Cena   | Wrap de pollo | · Tortillas Wrap   <br/> · Pollos   <br/> · Lechuga   <br/> · Sal
		- Tendrá un botón "Descargar en pdf" al pulsarlos pondra una lista de todos los ingredientes (únicos, es decir en el ejemplo anterior sólo debería aparecer en la lista sal una vez) de las comidas seleccionadas.
			- El formato será una tabla ordenada por "Supermercado"
			Id | nombre | Supermercado


- Segundo php "receta.php"

	- Quiero generar un fichero php que lea la receta del fichero receta.yaml que se le pasa como id para el requisito "Enlace a la receta, siempre será la pagina receta.php?id=(Id del plato)"
